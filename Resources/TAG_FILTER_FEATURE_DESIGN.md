# 标签筛选与固定标签设计文档

## 目标

让现有标签系统从“可标记”升级为“可导航、可筛选、可组织工作流”的入口，同时保持首页默认视觉极简。

本次设计聚焦两个页面：

- 术前页
- 术后页

不改动 `Patient` 数据模型，不引入新的患者字段；优先复用现有标签、颜色、最近使用和概览体系。

## 设计原则

- 默认态极简：首页只看到一条轻量横向标签栏，不出现复杂筛选面板。
- 能力完整版：点击 `更多` 后进入完整标签筛选视图。
- 高价值优先：先解决“快速只看某一类病人”。
- 当前上下文优先：术前页看到术前标签人数，术后页看到术后标签人数。
- 不做后台感：不加标签搜索，不做复杂条件组合。

## 功能范围

### 1. 列表页标签筛选

在术前页和术后页顶部加入一条横向可滚动标签栏，用于筛选当前页面列表。

默认展示：

- `全部`
- 当前选中标签
- 已固定标签
- 其余标签
- `更多`

说明：

- 标签栏位于搜索框下方、列表内容上方。
- 标签栏为单选筛选，不支持多标签组合筛选。
- 横向可滚动，允许直接选择更多标签。
- `更多` 按钮固定在右侧，不随标签滚动消失。

### 2. 完整标签面板

点击 `更多` 后，以 `bottom sheet` 形式展示当前页面全部可用标签。

面板能力：

- 展示当前 tab 下的全部标签
- 每个标签显示当前 tab 下的人数
- 点击标签后切换当前筛选
- 支持固定 / 取消固定
- 高亮当前选中的标签
- 不提供搜索框

### 3. 固定标签

用户可以将任意标签固定到顶部横向栏前部。

规则：

- 不限制固定标签数量
- 固定标签在横向栏中优先展示
- 固定标签在 `更多` 面板中优先排序
- 固定状态应持久化保存

## 信息架构

### 默认态结构

术前 / 术后页面顶部布局：

1. 第一行：标题 + 搜索框
2. 第二行：横向标签栏 + 固定在右侧的 `更多`
3. 第三部分：原有列表内容

### 默认态标签栏展示规则

顺序如下：

1. `全部`
2. 当前选中标签
3. 已固定标签
4. 其余标签按当前 tab 下使用人数降序

补充规则：

- 如果当前选中标签已经包含在固定标签或普通标签中，不重复显示。
- `全部` 永远放在最前。
- 当前选中标签不是 `全部` 时，需要保证可见。
- 默认态不显示标签人数。

### `更多` 面板展示规则

面板内标签排序：

1. 当前选中标签
2. 已固定标签
3. 其余标签按当前 tab 下使用人数降序

面板内每个标签项展示：

- 标签名称
- 当前 tab 下的人数
- 当前是否选中
- 当前是否固定

## 排序规则定义

### 使用数量的定义

“使用数量”指当前 tab 下，拥有该标签的患者人数。

示例：

- 在术前页中，`需追踪` 的数量 = 所有术前患者中带有 `需追踪` 标签的人数
- 在术后页中，`需追踪` 的数量 = 所有术后患者中带有 `需追踪` 标签的人数

不采用以下指标：

- 标签最近点击次数
- 历史使用次数
- 最近使用时间

原因：

- 当前人数最稳定
- 容易理解
- 与 `更多` 面板中展示的人数一致

### 空标签处理

- 若某标签在当前 tab 下人数为 0，则不在当前页面展示
- 即术前页只展示术前实际存在的标签，术后页同理

这样可以避免用户看到无效筛选项。

## 交互规则

### 默认态横向标签栏

- 点击 `全部`：清除当前标签筛选
- 点击普通标签：切换为该标签筛选
- 当前选中标签再次点击：不取消，维持选中
- 用户可左右滚动查看更多标签

视觉建议：

- 未选中标签：中性浅底或细描边
- 选中标签：使用该标签颜色
- `全部`：使用中性色体系，不使用标签色
- `更多`：中性按钮样式，弱于标签选中态

### `更多` 面板

- 点击某标签：应用该标签筛选并关闭面板
- 点击固定按钮：切换固定状态，不关闭面板
- 面板中应保留一个 `全部` 选项，便于清除筛选

建议每行交互拆分：

- 点击主区域：切换筛选
- 点击固定图标：固定 / 取消固定

避免单击一整行时同时触发两个动作。

### 与搜索的关系

标签筛选和文本搜索应同时生效。

最终列表条件：

- 先按当前 tab 过滤
- 再按文本搜索过滤
- 再按选中标签过滤

即搜索和标签是叠加关系，而不是互斥关系。

### 与多选模式的关系

建议：

- 进入多选模式后隐藏横向标签栏
- 或至少禁用其交互

推荐直接隐藏，原因：

- 多选模式的核心任务是批量操作
- 顶部再保留筛选入口容易造成注意力冲突
- 当前列表已被筛选后的结果仍然保留，不影响多选对象范围

## 数据与状态设计

### 页面状态

术前页 / 术后页各自维护：

- 当前搜索文本
- 当前选中标签：`String?`，`nil` 表示 `全部`
- `更多` 面板开关

### 持久化状态

新增一类轻量配置存储：

- 固定标签列表

建议按 tab 分开保存：

- 术前固定标签
- 术后固定标签

原因：

- 术前和术后的工作关注点可能不同
- 同一个标签在两个页面的重要性可能不同

推荐存储方式：

- `UserDefaults`

不建议存进 `Patient` 或 SwiftData 模型。

## 数据派生建议

建议抽出通用派生逻辑，避免术前页和术后页各写一套：

- 当前页面患者集合
- 当前页面可用标签集合
- 标签 -> 当前页面人数映射
- 横向栏显示标签列表
- `更多` 面板显示标签列表

可考虑新增一个轻量 helper，例如：

- `TagFilterStore`
- 或 `TagFilterSupport`

职责建议：

- 维护固定标签
- 生成排序后的标签列表
- 提供人数统计

不要把这部分逻辑塞进 `TagColorStore`，因为颜色存储和筛选行为不是同一职责。

## 建议实现拆分

### Phase 1

完成基础可用版本：

- 术前页加入横向标签栏
- 术后页加入横向标签栏
- 支持单标签筛选
- 支持 `更多` 按钮
- `bottom sheet` 展示全部标签和人数

### Phase 2

完成固定标签能力：

- 固定 / 取消固定
- 固定标签持久化
- 横向栏排序联动

### Phase 3

做结构整理与复用：

- 提取共享组件
- 提取共享派生逻辑
- 清理术前页与术后页重复代码

## 推荐组件拆分

建议新增的 UI 组件：

- `TagFilterBar`
- `TagFilterChip`
- `TagFilterSheet`
- `TagFilterRow`

建议新增的状态 / 工具层：

- `TagFilterStore`

## 验收标准

满足以下条件即视为完成：

- 术前页可通过顶部标签栏筛选当前列表
- 术后页可通过顶部标签栏筛选当前列表
- 默认态标签栏支持横向滚动
- `更多` 入口始终可见
- `更多` 面板展示当前 tab 下的全部标签及人数
- 可在 `更多` 面板中固定 / 取消固定标签
- 固定标签会回到顶部横向栏优先展示
- 固定标签状态在重启 app 后仍保留
- 搜索和标签筛选可叠加工作
- 进入多选模式后不会出现交互冲突

## 暂不纳入本次范围

以下内容明确不在本轮内：

- 多标签组合筛选
- 标签搜索框
- 概览页固定标签 section
- 标签自动 badge 行为扩展
- 标签驱动自动排序
- 新的系统语义标签设计

这些能力可在本功能稳定后继续扩展。

## 一句话结论

这次不是做“一个更复杂的标签管理器”，而是把标签真正变成术前 / 术后页面里的工作导航入口：默认极简，展开完整，行为统一，信息足够，但不过度打扰。
