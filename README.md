# S-Day

> 面向外科日常工作的 iPhone 原生患者清单工具。  
> 当前版本：`v0.3.0`

S-Day 不是完整 HIS，也不是通用表格，而是一个围绕“今天要处理哪些病人”设计的个人工作台。

它强调 4 件事：

- 录入要快
- 分组要清楚
- 标签要顺手
- 数据要始终留在本地

## 这个版本意味着什么

`v0.3.0` 代表当前产品已经从“能用的原型”进入“可以稳定带入日常工作流”的阶段。

这一版已经具备：

- 术前 / 术后双列表管理
- 持续录入与快速整理
- 标签筛选与批量操作
- 本地数据导入导出
- 隐私锁保护
- 从桌面快捷菜单直接进入新增病人

如果你希望的是一个轻量、离线、低摩擦的手术患者清单工具，这一版已经基本成型。

## 适合的使用场景

- 术前患者快速录入、排期和分组整理
- 术后患者持续追踪和标签筛查
- 病房、门诊、手术室之间的临时工作清单
- 需要完全本地保存、不依赖网络的个人使用场景

## 核心功能

### 1. 患者清单

- 术前、术后分开管理
- 按手术日期自动分组
- 支持未排期患者单独归组
- 支持组内拖动排序
- 支持分组折叠与展开
- 左滑病人可快速设日期、打标签或删除
- 右滑病人可直接进入多选模式

### 2. 快速录入

- 术前页顶部常驻“新病人”输入行
- 支持连续录入，回车即可提交
- 输入 `#标签` 会自动抽取为标签胶囊
- 已有标签支持联想补全
- 提交后可保持输入焦点，适合连续建档
- 支持桌面图标长按快捷菜单“添加新病人”
- 从快捷菜单进入时会自动打开术前页并聚焦输入框、弹出键盘

### 3. 标签系统

- 内置系统标签：`需追踪`、`收藏`
- 自定义标签会自动进入全局标签库
- 支持标签颜色自动分配
- 支持标签重命名、删除和颜色调整
- 支持记录最近使用标签
- 术前页、术后页都支持顶部标签筛选

### 4. 批量操作

- 多选后批量删除
- 多选后批量设置手术日期
- 多选后批量打标签
- 多选后批量复制导出文本
- 支持分组级整组勾选

### 5. 搜索与概览

- 术前页实时搜索
- 术后页实时搜索
- 概览页全局搜索全部患者
- 概览页显示术前 / 术后统计
- `需追踪` 患者数量会显示在 Tab badge 上

### 6. 隐私与数据安全

- 支持 Face ID / Touch ID / 设备密码解锁
- App 从后台返回时可自动重新鉴权
- 支持 JSON 导出
- 支持 JSON 导入并覆盖恢复
- 导入和清空数据都有显式文字确认，避免误操作
- 数据默认保存在本地，不依赖账号系统

### 7. 使用体验

- 日期标题支持今天 / 明天 / 昨天等相对显示
- 支持浅色、深色、跟随系统
- 多选模式下尽量保持列表稳定，减少跳动
- 关键操作配有触觉反馈和轻量 toast

## 技术栈

| 项目 | 说明 |
| --- | --- |
| 平台 | iOS |
| 语言 | Swift |
| UI | SwiftUI |
| 数据持久化 | SwiftData |
| 生物认证 | LocalAuthentication |
| 文件导入导出 | UniformTypeIdentifiers + `fileImporter` / `fileExporter` |

## 系统要求

- iOS 17.0+
- Xcode 16+ 可编译运行

## 本地运行

1. 用 Xcode 打开 [S-Day app.xcodeproj](S-Day%20app.xcodeproj)
2. 选择可用的签名团队
3. 选择模拟器或真机运行

默认情况下，所有数据仅保存在本地设备。
